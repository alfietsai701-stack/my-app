## 四、系統修正內容

**① 週日公休改為可設定（預約管理模組）**
新增 `lib/business-hours.ts`，公休改由 `Setting` 資料表的 `business_hours` 驅動，支援三種設定：每週固定公休、臨時公休日、特殊營業日（覆蓋固定公休）。`getAvailableSlots` 於公休日直接回傳空時段；預約前端 `BookForm` 改讀 `/api/book/config`，移除寫死的 `isWeekend`／「週日公休」字樣。後台「系統設定」新增「營業與公休設定」卡片可即時調整。預設維持週日公休，行為相容。

**② 第三方登入 Google／LINE（介面與使用者體驗）**
新增 OIDC 登入流程 `/api/auth/oauth/[provider]` 與 callback，沿用現有 JWT cookie（抽出 `lib/auth-session.ts` 共用簽發）。以 email 對應既有管理員帳號，不自動建帳。登入頁新增「使用 Google／LINE 登入」按鈕（以環境變數開關），並加上第三方登入失敗的友善訊息。

**③ 顧客表格對齊修正（介面與使用者體驗）**
`CustomersClient` 表頭「電話」原為靠右、內容靠左造成錯位；已統一「姓名／電話」欄位表頭與內容皆靠左，數值欄位（年齡、預約次數）維持靠右。

**④ LINE 綁定／自動帶入／前一日提醒（顧客資料管理・預約・通知）**
- 綁定與自動帶入：透過 LIFF 取得顧客 `lineUserId`，新增 `/api/book/customer` 依此查詢老客資料，於預約表單自動帶入姓名／電話／Email，並顯示「已帶入上次資料」提示。
- 預約成功通知：維持既有 LINE 推播確認訊息。
- 前一日提醒：新增 `/api/cron/reminders`，由 `vercel.json` 設定每日台灣時間 10:00 觸發，對「明天有預約且已綁定 LINE」的顧客自動推播提醒，降低忘記機率。

**⑤ 顧客備注標籤化（顧客資料管理）**
`Customer` 新增 `tags` 欄位並提供預設標籤（VIP、新客、熟客、敏感肌…）與自訂標籤。顧客 API 支援以標籤新增／更新／篩選；`CustomersClient` 於列表、詳情顯示標籤，並提供標籤篩選與編輯 UI。

**⑥ 首次顧客測驗推薦（預約與管理模組）**
預約頁新增「預約／測驗推薦」切換。測驗以 3 題（改善方向、膚況偏好、預算）依關鍵字與價格區間為現有服務評分，推薦最合適的前三名方案，可一鍵帶入預約流程。

## 六、AI 建議事項（由 Claude 依本報告分析）

**優先處理建議**
1. **上線前務必備份資料庫**，再執行 `prisma db push` 新增 `tags` 欄位（不可逆風險最高）。
2. **設定 `CRON_SECRET`** 並確認 Vercel Cron 生效，否則前一日提醒不會自動發送——這是第 4 項高優先問題的關鍵一環。
3. **第三方登入採 email 對應**：請先確認各管理員的 Google／LINE email 與後台帳號一致，並在 LINE Developers 另建 LINE Login channel（與 Messaging API channel 不同）。
4. 於 staging 完成一次端到端實測（公休擋預約、LINE 帶入、提醒推播、標籤篩選、測驗推薦）。

**長期優化建議**
1. **提醒與排程可觀測性**：為 `/api/cron/reminders` 加上發送記錄（成功／失敗數）與失敗重試，並保留推播 log。
2. **速率限制外部化**：登入與時段快取目前為單機記憶體，流量上升後建議改用 Redis／KV。
3. **標籤治理**：待標籤量增加，可將預設標籤改為後台可維護清單，並加上標籤使用統計。
4. **測驗精準度**：目前為關鍵字比對，累積資料後可依實際成交項目調整權重，或改為後台可編輯的題庫與對應表。
5. **自動化測試**：針對時段運算、公休判斷、標籤 sanitize 等純函式補單元測試，避免日後回歸。
6. **通知渠道整合**：預約成功／提醒／異動可統一為訊息模板管理，方便未來調整文案與多語系。
